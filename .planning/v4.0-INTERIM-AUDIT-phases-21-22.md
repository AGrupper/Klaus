## Cross-phase integration (21 ↔ 22 seam)

Integration checker verdict: **wired, with two warnings.** The core seam is functionally
complete — both `{training_profile}` (phase 21) and `{coaching_guide}` (phase 22) resolve in
the same `render_smart_system` pass without clobbering each other; no literal placeholders
survive; the E2E coaching query reaches the brain with both Tier A targets and coaching
knowledge in context (live-verified SC-3). Two defects found, both confirmed against live code:

### WRN-01 — double `{training_profile}` substitution (higher severity)
`prompts/smart_agent.md` has `{training_profile}` at line 9 (live) **and** line 87 (inside the
prose "The `{training_profile}` block injected above…"). `core/main.py:428` uses
`str.replace`, which replaces **all** occurrences — so a seeded profile injects the full
~28-line block at both locations: a duplicated Tier A block (+~2627 chars) and a corrupted
mid-sentence at line 87. Does **not** break the flow (data is present; Klaus was live-verified
working), but degrades prompt quality and corrupts the instruction that explains the Tier A/B
contract. **Affects PLAN-01, COACH-02, COACH-07.** One-line fix: de-literalize line 87.

### WRN-02 — `update_plan` visible to worker (lower severity)
`update_plan` is brain-direct (`SMART_AGENT_DIRECT_TOOLS`) but missing from the
`WORKER_TOOL_SCHEMAS` exclusion set at `core/tools.py:911`, unlike its sibling
`update_training_profile`. The worker can see/call it. No bad write observed — the brain
direct-call path takes precedence — but it's a latent contract inconsistency. **Affects
PLAN-03.** One-line fix: add `"update_plan"` to the exclusion set.
