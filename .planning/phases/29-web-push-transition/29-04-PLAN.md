<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server → push service (APNs/FCM) | encrypted payload + VAPID JWT crosses to a third-party gateway |
| server → Secret Manager | VAPID private key loaded into process memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-05 | Information disclosure | VAPID private key in memory | mitigate | Loaded lazily from Secret Manager, cached in a module var, never logged, never returned in any tool/route response |
| T-29-06 | Spoofing | push payload forgery | accept | Payloads are RFC 8291 end-to-end encrypted; forgery requires the VAPID private key + per-subscription keys (handled by pywebpush) |
| T-29-07 | Denial of service | blocking the event loop on slow sends | mitigate | webpush carries timeout=10; callers invoke send_push_to_all via run_in_executor (enforced in Plans 06/08) — weekly-review-500 class |
</threat_model>
