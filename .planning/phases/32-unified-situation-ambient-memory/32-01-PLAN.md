Establish the MEM-05 token-budget guard: add the `tiktoken==0.13.0` dependency and a
deterministic, network-free test that counts the *maximal* rendered triage prompt plus
the tick-brain completion budget against Groq's verified 8,000-token per-request ceiling
for `openai/gpt-oss-120b`.

metered Gemini) was a per-request admission-control failure. This guard prevents the same
class of bug via prompt bloat as Phase 32 adds `conversation_tail` and `training_reality`
render slots. Research warns headroom is ~300 tokens — an estimate would false-pass or
false-fail at exactly the margin that matters, so the real tokenizer is mandatory.

wires the new triage render slots (Plan 07 re-verifies it stays green with slots populated).
