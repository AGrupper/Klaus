actually cached and only truly per-request content stays uncached — the structural
prerequisite for placing the three new volatile blocks (MEM-01 "Things you remember",
MEM-02 continuity tail, MEM-04 training_reality) without silently killing cache reads.

READ (Anthropic hashes per marked block, cumulatively — string-ordering within one block
does nothing). The CONTEXT.md "place new blocks after the stable prefix" is INSUFFICIENT.
The fix is a real SECOND content block, not positional ordering (Pattern 5 / Pitfall 1).

emits two blocks; all three `render_smart_system` callers pass the tuple through.
