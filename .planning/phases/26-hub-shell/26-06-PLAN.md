<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SPA → `/api/*` | Every data call crosses here; `apiFetch` attaches the httpOnly session cookie via `credentials:'include'` |
| unauthenticated visitor → app | The route guard is a UX gate only — the real control is the server-side `require_hub_session` on every `/api/*` route |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-06-01 | Elevation of Privilege | Client-side route guard treated as the security boundary | mitigate | The guard is UX only; every `/api/*` route enforces `require_hub_session` server-side (26-03); a bypassed guard still gets 401 from the backend |
| T-26-06-02 | Information Disclosure | Session cookie not sent → silent unauth, or sent cross-origin | mitigate | `credentials:'include'` + same-origin architecture (no CORS) + `SameSite=Strict` cookie (26-03); 401 redirects to sign-in (Pitfall 5) |
| T-26-06-03 | Tampering | Bundling a third-party testing-library of unknown provenance | accept | `@testing-library/react`/`jest-dom` are canonical, dev-only (never shipped in the PWA bundle); noted in SUMMARY |
| T-26-06-04 | Information Disclosure | Stale cached shell after deploy masking a security fix | mitigate | Network-first `index.html` SW from 26-01 (HUB-03) — out of this plan's scope but the shell relies on it |
</threat_model>
