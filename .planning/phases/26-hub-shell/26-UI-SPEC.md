## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip padding |
| sm | 8px | Message bubble inner padding, timeline item row gaps |
| md | 16px | Default element spacing, sidebar icon vertical rhythm |
| lg | 24px | Section padding within cards and rails |
| xl | 32px | Panel-level gaps (timeline ↔ glance rail) |
| 2xl | 48px | Major section breaks (header ↔ first timeline item) |
| 3xl | 64px | Page-level vertical spacing (desktop) |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements on phone (iOS HIG). Applied to bottom tab items, timeline row tap targets, and chat send button.
- Collapsible docked chat panel: collapses to 48px header strip (matches 2xl token).

**Source:** 8-point scale default per GSD template. Touch-target exception from iOS HIG; CONTEXT.md D-12 implies iPhone is primary phone target.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 13px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.15 |

**Notes:**
- Body (16px/400/1.5): chat message text, timeline event titles, glance rail values.
- Label (13px/500/1.4): timestamps, metadata rows (macros, sleep stats, weather one-liner), badge numbers, sidebar icon labels on desktop, "Week N of 16" block context chip.
- Heading (20px/600/1.2): timeline date header ("Today — Saturday 14 June"), section headings within the glance rail.
- Display (28px/600/1.15): reserved for the "Klaus" wordmark in the sidebar and the chat window title on phone.
- Maximum 2 weights in use: regular (400) for all body copy; medium (500) for labels only where 400 is too light at 13px; semibold (600) for all headings and display. Medium (500) is the only exception to the 2-weight rule and is strictly confined to 13px label role.

**Source:** GSD template defaults. Sizes chosen to match a personal-productivity PWA — no decorative sizes. Confirmed consistent with Tailwind v4 prose and readable on iPhone screens.

## Color

The design spec's PWA manifest declares `theme_color: '#000000'` and `background_color: '#000000'`. The visual intent is a **dark theme** — this is the locked color direction.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0A0A0A | Page background, sidebar background, main surface |
| Secondary (30%) | #1A1A1A | Cards (timeline event cards, glance rail card, chat panel background, bottom tab bar) |
| Accent (10%) | #6366F1 (indigo-500) | Reserved for specific elements only — see list below |
| Destructive | #EF4444 (red-500) | "Sign out everywhere" confirmation button; send-error state on chat message bubble |
| Text primary | #F9FAFB | All body copy and headings on dark surfaces |
| Text secondary | #9CA3AF | Timestamps, metadata, dimmed past timeline items (D-04), placeholder text |
| Surface border | #2A2A2A | Dividers between timeline items, card borders, separator lines |
| Online / success | #22C55E (green-500) | Message "sent" checkmark, "connected" indicator |

**Accent (#6366F1 indigo-500) is reserved ONLY for:**
1. The unread badge on the Klaus bottom tab and chat panel header
2. The active/selected state of the bottom tab icon (phone) and sidebar nav icon (desktop)
3. The chat send button background
4. The now-line marker on the Today timeline (D-04)
5. The "Install Klaus" banner call-to-action button (D-12)

Accent is explicitly NOT used for: general links, hover states, timeline event backgrounds, section headings, or nutrition macro labels.

**Dimmed past items (D-04):** Past timeline items use `opacity: 0.45` over their normal card styles — no separate background color. The now-line (accent #6366F1) visually separates past from upcoming.

**Skeleton / loading state:** Use `#1F1F1F` animated shimmer (CSS `animate-pulse` on a `#1F1F1F` background over `#0A0A0A`) for in-flight API data (HUB-03).

**Offline indicator:** A 4px top border strip in `#F59E0B` (amber-500) on the offline banner (not reserved for any other use).

**Source:** PWA manifest `theme_color: '#000000'` locks dark theme (RESEARCH.md Pattern 4). Indigo accent chosen because it reads clearly on near-black backgrounds at the 10% usage level and is a standard Tailwind v4 color requiring no custom token. Green/red for semantic states are standard. Text colors follow Tailwind dark-mode conventions.

## Interaction Contracts

### Today Timeline

- **Auto-scroll on open:** `useEffect` scrolls the `NowLine` element into view with `scrollIntoView({ behavior: 'smooth', block: 'center' })` on initial mount.
- **Refresh triggers:** `useQuery` with `refetchOnMount: true` + `refetchOnWindowFocus: true`. No timer polling. Pull-to-refresh on phone via touch gesture triggering `queryClient.invalidateQueries({ queryKey: ['today'] })`.
- **Past item dimming:** Apply `opacity-[0.45]` Tailwind class to any `TimelineItem` whose end time is before `Date.now()`.
- **Loading state:** `Skeleton` components replace each timeline section during initial fetch.
- **D-06 placeholders (not skeletons):** When API returns `null` for `coach_note` or `garmin`, render `PlaceholderCard` with fixed text (see Copywriting Contract). No shimmer — these are stable data-not-yet-available states.

### Chat

- **Optimistic send:** Message appears immediately as `status: 'sending'`. On server ACK → `status: 'sent'` (green checkmark). On error → `status: 'error'` (destructive red, retry tap).
- **Polling:** `refetchInterval: 2500` only while `ChatWindow` is mounted and visible. Pauses on tab blur (TanStack default).
- **Scroll-to-bottom on new Klaus message:** Auto-scroll if the user was already at the bottom when the new message arrives. Do not scroll if user has scrolled up to read history.
- **Unread badge clear:** When `ChatWindow` bottom is visible (IntersectionObserver on last message), set `localStorage.last_seen_seq = messages.length`.
- **Send shortcut:** Desktop: `Enter` sends, `Shift+Enter` adds a newline. Phone: send button only.

### PWA Install Banner (D-12)

- Show only if: `isIOS && !isStandalone && !localStorage.getItem('install-banner-dismissed')`.
- Position: fixed bottom, above BottomTabs on phone; above viewport bottom on desktop.
- Dismiss: tapping X sets `localStorage.install-banner-dismissed = '1'` and hides permanently.

### Sign Out / Sign Out Everywhere (D-02)

- "Sign out" link in sidebar footer (desktop) and in a settings sheet (phone): clears session cookie, redirects to SignInPage. No confirmation dialog — this is low-stakes on a personal device.
- "Sign out everywhere": behind a confirmation modal (see Copywriting Contract). Calls `/api/auth/revoke-all`, then clears local session and redirects to SignInPage.

## Responsive Layout Contract

### Desktop (>= 768px / Tailwind `md:`)

```
┌──────────────────────────────────────────────────┐
│ [64px sidebar] │ [Today Timeline — flex-1] │ [Glance Rail 280px] │ [Chat Panel 360px collapsible] │
└──────────────────────────────────────────────────┘
```

- Sidebar: 64px wide, full height, icon-only nav with lucide-react icons, active icon in accent.
- Timeline: `flex-1`, min-width 0, scrollable.
- Glance rail: 280px fixed, non-scrolling header section (nutrition totals + future tasks/streaks).
- Chat panel: 360px wide, collapsible to 48px strip via header chevron button. Chat panel is the rightmost column.

### Phone (< 768px)

```
┌─────────────────────┐
│ [Full-width content] │
│                     │
│                     │
├─────────────────────┤
│ [Bottom Tab Bar 64px] │
└─────────────────────┘
```

- Bottom tab bar: 64px height, 5 tabs (Today · Tasks · Klaus · Habits · Health). Klaus center.
- Content area fills remaining height.
- Chat is full-screen when Klaus tab is active.
- Glance rail content surfaced as a horizontal scroll strip below the timeline header on phone (not a separate column).

**Breakpoint:** Tailwind `md` (768px). No intermediate breakpoints needed for this phase.
