<objective>
Stand up the greenfield frontend toolchain and wire FastAPI to serve it, without breaking any existing route. This is the foundation every later Phase 26 plan builds on.

Purpose: The Klaus repo has never had a frontend. This plan creates the Vite + React + TypeScript + Tailwind v4 + vite-plugin-pwa scaffold, the multi-stage Dockerfile (Node build stage → copy `dist` into the existing `python:3.11-slim` image, single uvicorn worker preserved), and the `SPAStaticFiles` catch-all mounted as the absolute last route in `web_server.py` so it never shadows `/api/*`, `/cron/*`, `/internal/*`, `/trigger/*`, `/telegram-webhook`, or `/health` (HUB-04). Establishes HUB-03's service-worker caching contract (network-first `index.html`, cache-first hashed assets).

Output: `frontend/` project that builds to `frontend/dist`, a multi-stage `Dockerfile`, the SPA mount + a regression test proving existing routes survive, and the dark-theme design tokens in `frontend/src/tokens.ts`.
</objective>
