<objective>
Close verification gap 1 (DIR-05 / SC-5 / D-16): make the `superseded_by` audit link
chat-reachable. `StandingDirectiveStore.supersede()` already exists, is unit-tested, and
correctly writes `status="superseded"` + `superseded_by=<new_id>` on the old doc — but it
has ZERO production callers. The `set_standing_directive` tool exposes no supersede/old_id
param, no supersede tool exists, and `prompts/smart_agent.md`'s persona-conflict rule tells
the brain to new-set + `cancel_standing_directive` the old one, producing `status="cancelled"`
with no link.

Purpose: DIR-05 requires that a persona conflict be resolved AT CAPTURE by recording the
answer as a refined directive WITH a `superseded_by` link on the old one (D-16). Cancel-and-
recreate loses that durable audit link. This gap makes the existing `supersede()` reachable
from the chat capture path.

Output: an optional `supersedes` param on `set_standing_directive` whose handler calls
`store.supersede()` after `add()`, an updated persona-conflict prompt rule, and a test
asserting a chat-driven persona-conflict capture writes `superseded_by` (not `cancelled`).
</objective>
