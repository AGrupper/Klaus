BlockStore — collection "training_blocks", doc_id = "{YYYY-MM-DD}_{label_slug}" (e.g. "2026-06-21_aerobic_base"):
  block_id: str (== doc id) · label: str · start_date: str YYYY-MM-DD · end_date: str YYYY-MM-DD
  focus_facets: list[str] · weekly_split_override: dict|None · status: "active"|"complete"|"abandoned"|"pending"
  notes: str · benchmark_due: bool · created_at/updated_at: firestore.SERVER_TIMESTAMP

  get_current() RESOLUTION SEMANTICS (D-01 — the critical contract):
  get_current() resolves the active block by DATE RANGE — it returns the single seeded block whose
  start_date <= today <= end_date. Resolution is automatic and date-driven: because all 4 blocks are
  seeded at cycle start (D-01), the correct block surfaces as time advances WITHOUT any manual
  start_block call. The `status` field is bookkeeping metadata (set by start_block/end_block for the
  current_block_id FK and benchmark housekeeping) — get_current()'s correctness MUST NOT depend on it.
  Returns None when today is before the earliest start_date or after the latest end_date (pre/post cycle).

BenchmarkStore — collection "benchmarks", doc_id = "{YYYY-MM-DD}_{facet}" (e.g. "2026-07-18_bench_press_1rm"):
  date: str · facet: str · value: float · unit: "kg"|"reps"|"sec_per_km" · block_id: str (FK)
  notes: str · updated_at: firestore.SERVER_TIMESTAMP
