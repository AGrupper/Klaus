# Phase 30.5 Plan 02: Firestore Storage Foundation Summary

**Cache-aware, per-purpose-cost LLM usage metering, a TTL-cached UserProfileStore, and a per-date cost-tripwire suppression store, all additive to `memory/firestore_db.py` with zero migration.**

## Performance

- **Duration:** ~20 min
- **Tasks:** 3
- **Files modified:** 2

## Accomplishments
- `LLMUsageStore.record()` now accumulates `total_cache_read_tokens`, `total_cache_write_tokens`, and a `{purpose}_cost_usd` per-purpose cost driver, alongside a new `summary_for_date(date_str)` never-raises reader — the storage layer Plan 03 (metering) and Plan 05 (daily tripwire) will consume.
- `UserProfileStore.load()` is now served from the shared module TTL cache (600s), mirroring `SelfStateStore`'s exact pattern; `update()` invalidates on the success path only.
- New `CostTripwireLogStore` provides a per-date once-only suppression gate (`already_fired`/`mark_fired`) for the daily cost-tripwire alert (BRAIN-04), ready for Plan 05's heartbeat wiring.

## Task Commits

Each task was committed atomically:

1. **Task 1: Extend LLMUsageStore — cache-token fields, per-purpose cost, per-date summary** - `bbba905` (feat)
2. **Task 2: TTL-cache UserProfileStore.load() (BRAIN-07)** - `5f8bae9` (feat)
3. **Task 3: Add per-date cost-tripwire suppression store (BRAIN-04 gate)** - `6df5c04` (feat)

_No separate plan-metadata commit — this SUMMARY.md is committed as part of the worktree wave; the orchestrator handles final STATE.md/ROADMAP.md updates after merge._

## Files Created/Modified
- `memory/firestore_db.py` - `LLMUsageStore.record()` extended with `cache_read_tokens`/`cache_write_tokens` kwargs (default 0) + `{purpose}_cost_usd` Increment field + updated schema docstring; new `LLMUsageStore.summary_for_date()`; `UserProfileStore.__init__`/`load()`/`update()` gained the TTL-cache pattern; new `CostTripwireLogStore` class (collection `cost_tripwire_log`) with `already_fired()`/`mark_fired()`.
- `tests/test_firestore_db.py` - Added `TestLLMUsageStoreCacheAndPerPurposeCost` (5 tests), `TestUserProfileStoreCache` (3 tests), `TestCostTripwireLogStore` (5 tests) — 13 new tests total.

## Decisions Made
- Per-purpose cost (`{purpose}_cost_usd`) was added in addition to the existing per-purpose call counters (`{purpose}_calls`) so the tripwire's "top 2-3 cost drivers by purpose" reflects actual USD spend rather than call volume (per 30.5-RESEARCH.md Open Question 2).
- `CostTripwireLogStore` deliberately does not wire into `core/heartbeat.py` in this plan — that wiring belongs to Plan 05, keeping this plan's scope limited to the storage layer.
- Existing `LLMUsageStore.record()` 5-positional-arg call site in `core/llm_client.py` was left untouched (out of scope for this plan); the new kwargs default to 0 so it keeps working unchanged.

## Deviations from Plan

None - plan executed exactly as written. All three tasks matched their `<action>` blocks precisely; no Rule 1-4 auto-fixes were needed.

## Issues Encountered
- The system `python` on PATH resolves to Homebrew's global Python 3.14 (per project memory, prod/dev must use 3.13 — 3.14 has known grpc/protobuf GC segfault risk). Ran all verification through the project's `.venv` (Python 3.13.12) instead of the bare `python3` on PATH. No code changes needed; noting for awareness since a bare `python3 -m pytest` from a fresh shell will otherwise silently skip network-dependent test modules (fastapi/psycopg2/pywebpush not installed in the global interpreter) and could mask real failures if used for verification.

## User Setup Required
