## Decisions Made

- **Cron list source of truth:** used CLAUDE.md §5 (explicitly cited by the task as "the truthed list") rather than STATE.md's cron table, which itself still incorrectly lists retired `proactive-alerts`/`reflect` jobs as active — CLAUDE.md is the accurate live-infrastructure record.
- **Tool categorization implementation:** used explicit `name -> category` / `name -> short-label` dicts (covering all 56 currently-registered tools) rather than deriving categories generically from name prefixes, for label quality and correctness; any future tool not yet added to the dict falls back to an "Other" category with its full name, so it's never silently dropped from the manifest, satisfying the "introspection can never drift" requirement even for the display layer.
- **Measurement script scope:** `scripts/measure_prompt_tokens.py` intentionally does NOT instantiate `AgentOrchestrator` (which needs live Firestore/Google credentials) — it duplicates the minimal substitution logic from `render_smart_system`, using real file content for the stable/slimming-target sections (self_md, coaching_guide, smart_agent.md) and fixed placeholder text for the volatile per-request sections (self_state, journal_digest, training_profile, today_date, current_time), so the "before" vs "after" delta is attributable entirely to this plan's two changed files.
- **API key resolution order:** the research doc's code example assumes `SMART_AGENT_API_KEY` holds an Anthropic key, which is only true *after* Plan 06's brain flip. Since Plan 04 runs before Plan 06 in this wave sequencing, the script checks `ANTHROPIC_API_KEY` first, then `SMART_AGENT_API_KEY` (if `SMART_AGENT_BACKEND=anthropic`), then `SMART_AGENT_FALLBACK_API_KEY` (if `SMART_AGENT_FALLBACK_BACKEND=anthropic` — true today, since the current `claude-haiku-4-5` fallback already uses an Anthropic key). This makes the script work unchanged both before and after the flip.
