# Phase 30.5: Brain Upgrade (Sonnet 5) - Context

**Gathered:** 2026-07-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Klaus's smart brain moves from `gemini-3.5-flash` to `claude-sonnet-5` — with Anthropic
prompt caching active (explicit 1h TTL), truthful cache-token metering in LLMUsage, a
heartbeat daily-spend tripwire, a tick-brain fallback decoupled from `SMART_AGENT_*`
(deployed and verified BEFORE the model flip), a measurably slimmer always-on prompt
(compact SELF.md + light `smart_agent.md` de-prescription pass), and TTL-cached
`UserProfileStore` reads. Requirements BRAIN-01..07. No new capabilities — directives,
ambient memory, and occasion changes belong to Phases 31–33.

</domain>

<decisions>
## Implementation Decisions

### Locked upstream (approved plan + review amendments — do not re-litigate)
- Brain = `claude-sonnet-5`; `gemini-3.5-flash` flips to inline brain fallback (BRAIN-01)
- `TICK_BRAIN_FALLBACK_*` env decoupling deploys BEFORE the brain flip (BRAIN-03)
- Prompt caching with explicit `ttl: "1h"` (not the 5-min default — the 20-min tick cadence exceeds 5 min)
- Tripwire alerts, never blocks — "measure everything, cap nothing" stands
- Worker (`deepseek-v4-flash`) untouched this phase; retirement verdict is Phase 35 HARD-04
- Slimming measured with the real Sonnet-5 tokenizer via `count_tokens`, never char estimates

### Spend-tripwire experience (BRAIN-04)
- **D-01:** Alert **once per day** — first heartbeat after midnight computes yesterday's total; if over threshold, alert once and repeat-suppress for that date (mirror the existing OutreachLog per-date suppression pattern). No hourly nagging, no same-day escalation.
- **D-02:** Threshold `KLAUS_DAILY_COST_ALERT` defaults to **$5**, env-tunable without a deploy.
- **D-03:** Message is **Klaus-summarized**: total, top 2-3 cost drivers by purpose, cache-hit rate, plus Klaus's own read on whether it looks like normal usage or something broken. (BRAIN-04's per-purpose breakdown satisfied narratively, not as a table.)
- **D-04:** Alert text is **brain-composed** — one Sonnet call frames deterministically-computed numbers (the model never computes the numbers, only phrases them); deterministic plain-text template fallback if the compose call fails.

### Prompt slimming boundaries (BRAIN-06)
- **D-05:** **Untouchable:** the JARVIS/C-3PO persona voice AND the Tier A/B data-presence (no-fabrication) contract (`prompts/smart_agent.md` ~lines 167–201) stay verbatim. Both are battle-tested against real regressions.
- **D-06:** De-prescription is a **light pass** this phase: soften hard ALWAYS/NEVER imperatives that could over-fire on Sonnet's more literal instruction-following, delete instructions that existed only as Gemini hand-holding, drop redundancy. Behavior-shaping sections wait for the Phases 31–33 deep rewrite (avoid double-rewriting).
- **D-07:** SELF.md compaction = **grouped one-liners**: tools collapse to category lines ("Calendar: list/create/free-busy/delete…"), crons to one summary line. Klaus keeps a working capability map; full detail on demand via self-inspect tools. `core/self_manifest.py` generator changes, so it can never drift.
- **D-08:** Slimming target is **directional + measured** — acceptance is "measurably smaller via `count_tokens` on the real model, before/after numbers recorded". No fixed token threshold; do not cut past D-05's boundaries to hit a number.

### Thinking & output budget (BRAIN-05)
- **D-09:** **Let it think** — default adaptive thinking everywhere on the Anthropic path; the model calibrates per-turn effort. No effort-level tuning, no per-path thinking config. This IS the brain upgrade; cost is measured (tripwire), not capped.
- **D-10:** **Quality over speed** — no latency-motivated thinking constraints; `LLM_TIMEOUT_SECONDS` (120s) remains the only backstop and is re-validated live per BRAIN-05.
- **D-11:** `max_tokens` policy = **raise the Anthropic-path default to a generous ceiling (~16K)** so thinking + reply never truncate; keep deliberate small ceilings untouched (tick-brain 2048 is Groq admission control — do not touch). Ceilings are free until used; no per-purpose table.
- **D-12:** **Fallback disclosure:** when a turn is served by the Gemini fallback, Klaus appends a subtle one-liner ("running on backup reasoning at the moment, Sir" — in his voice, not this literal string). User explicitly chose self-aware transparency over silent fallback. Also logged in LLMUsage as today.

### Cutover & rollback (BRAIN-01/03)
- **D-13:** **Verify-then-flip same-day:** deploy the `TICK_BRAIN_FALLBACK_*` decoupling, force a Groq failure and confirm the fallback logs `gemini-3.5-flash` (never a Claude model), then ship the brain flip the same day. Sequencing is about verified order, not calendar spacing.
- **D-14:** Post-flip validation = **live checklist** on production: one Telegram turn, one hub turn, one tool-heavy coaching question, one forced Anthropic failure (confirm Gemini fallback + D-12 disclosure line), and next-day LLMUsage-vs-Anthropic-console comparison (~10% tolerance per BRAIN-02). No pre-flip canary harness.
- **D-15:** Rollback is **Amit's judgment call** — no predefined auto-rollback thresholds; he watches cost, quality, and personality for the first days. Rollback mechanism = env-var flip back to Gemini.
- **D-16:** **Haiku stays as tertiary fallback only:** chain is Sonnet → Gemini → `claude-haiku-4-5` (config-level, reuses the existing fallback path). Do NOT build cheap-task routing to Haiku this phase — recorded as the leading candidate for Phase 35's HARD-04 worker verdict (native Anthropic tool format, would drop provider count to two if a cheap tier survives).

### Claude's Discretion
- Cache breakpoint placement within the system prompt (stable prefix vs daily-changing `{self_state}`/`{journal_digest}` sections) — pure implementation, guided by the 1h-TTL decision and cache-hit-rate measurement.
- LLMUsage schema additions for cache-read/cache-write tokens and `compute_cost` signature changes.
- Pricing-table handling of Sonnet-5 intro pricing ($2/$10 through 2026-08-31 vs $3/$15 standard) — needs a dated note/mechanism, Claude picks the shape.
- `UserProfileStore` TTL-cache wiring (BRAIN-07) — the `_READ_CACHE` pattern already exists in `memory/firestore_db.py` (lines ~61–86); apply it.
- How exactly to force the Groq failure for D-13 verification (bad key, injected error, etc.).

### Established Patterns
- All model/backend strings are env-driven (`SMART_AGENT_*`, `TICK_BRAIN_*` + `_BACKEND`/`_API_KEY`/`_BASE_URL`) — the flip and the decoupling are env changes plus code that reads the new `TICK_BRAIN_FALLBACK_*` vars.

- Deploy gotcha (memory `klaus_web_push_gotchas`): `deploy.yml --set-env-vars` **clobbers out-of-band Cloud Run env vars** — new env vars (`TICK_BRAIN_FALLBACK_*`, `KLAUS_DAILY_COST_ALERT`) must be added to deploy.yml, not set only via console, or the next deploy wipes them.
