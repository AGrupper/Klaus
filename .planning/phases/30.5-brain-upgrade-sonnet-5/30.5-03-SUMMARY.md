# Phase 30.5 Plan 03: Anthropic Prompt Caching + Cost-Aware Metering Summary

**claude-sonnet-5 dated intro/standard pricing with 0.1x/2.0x cache multipliers, 1h-TTL `cache_control` on the Anthropic system prompt, cache-token extraction threaded into `LLMUsageStore.record`, and a 16K `MAX_TOKENS` default for adaptive-thinking headroom — zero forbidden params on the smart path.**

## Performance

## Accomplishments
- `core/pricing.py` now prices `claude-sonnet-5` with a dated intro ($2/$10 per 1M through 2026-08-31) vs standard ($3/$15) mechanism, and `compute_cost` accepts `cache_read_tokens`/`cache_write_tokens` kwargs (default 0, fully source-compatible with every existing caller) priced at 0.1x and 2.0x the active input rate respectively.
- `core/llm_client.py`'s `_AnthropicBackend.chat` sends `system` as a `cache_control: {"type": "ephemeral", "ttl": "1h"}` content-block list (only when `system` is non-empty), extracts `cache_read_tokens`/`cache_write_tokens` from `response.usage` defensively, and the `LLMClient.chat` metering block threads both fields into `compute_cost(...)` and `LLMUsageStore.record(...)` — the whole block remains inside its bare `try/except Exception` so metering never raises.
- `MAX_TOKENS` raised from 4096 to 16000 (D-11) so Sonnet 5's default-on adaptive thinking plus the visible reply never truncates; tick-brain's explicit 2048 ceiling is untouched since it always passes `max_tokens` directly.
- `requirements.txt` anthropic floor bumped to `>=0.99,<1.0` (installed 0.105.2 already satisfies it — no install needed).
- Verified via grep + a dedicated test that no current or new call site sends `temperature`/`top_p`/`top_k`/`thinking` on the Anthropic smart path.

## Decisions Made
- Kept `claude-sonnet-5` as a literal `MODEL_PRICING` dict key (rather than omitting it and resolving purely via a function) so the plan's stated acceptance criteria — `MODEL_PRICING contains a claude-sonnet-5 entry`, `grep -n "claude-sonnet-5" core/pricing.py` present, and the model-entry-count test (now asserting `== 7`) — are all literally satisfied, while `compute_cost` internally routes through `_pricing_for()` -> `_sonnet_5_pricing()` so the actual priced rate is always the live dated rate rather than whatever was captured in the dict at import time.
- Cache multipliers are computed against the resolved tier's `input` rate (not a hardcoded intro/standard constant), so cache pricing automatically tracks the correct tier across the 2026-08-31 cutover with no additional code change needed at that date.
- Confirmed (via grep of `core/main.py` and a dedicated `test_anthropic_backend_no_forbidden_params_on_smart_path` test) that the smart path never passes `temperature`, and added `test_anthropic_backend_temperature_still_gated_when_explicitly_passed` to lock in that the existing guard still works for callers (e.g. the inline Haiku fallback path) that do pass it explicitly.
