## Spacing Scale

Declared values (multiples of 4 only) — inherited and extended from Phase 26:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip inner padding, row gaps between task items |
| sm | 8px | Task card inner padding gaps, row spacing within lists |
| md | 16px | Default element padding, task card horizontal padding |
| lg | 24px | Section padding within task list view, sort/group control row |
| xl | 32px | Panel-level vertical gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing (desktop) |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (iOS HIG). Applied to task row tap targets, FAB, quick-add input, complete checkbox, edit/delete row actions, sort control buttons, and list sidebar items.
- FAB (floating action button on phone): 56px diameter (standard mobile FAB size — larger than touch-target minimum for affordance and glanceability on a tight bottom-tab layout).
- Undo toast: fixed 52px height to accommodate two lines of text at label size with 8px vertical padding.
- Quick-add token chip badges: 24px height (4px vertical padding + 16px label).

**Accent (#6366F1 indigo-500) is reserved ONLY for (Phase 27 additions in bold):**
1. The unread badge on the Klaus bottom tab and chat panel header (inherited)
2. The active/selected state of the bottom tab icon (phone) and sidebar nav icon (desktop) (inherited)
3. The chat send button background (inherited)
4. The now-line marker on the Today timeline (inherited)
5. The "Install Klaus" banner call-to-action button (inherited)
6. **The FAB button background on phone (quick-add)**
7. **The active/selected list item indicator in the Lists sidebar (desktop)**
8. **The "Due today" band left-border accent stripe on the Today timeline**

Accent is explicitly NOT used for: general task row backgrounds, hover states, priority chips, due date chips, or recurrence labels.

**Priority color system (separate from the main palette — task-specific semantic chips):**
- High priority: `#F87171` (red-400 text on `#2A1A1A` dark-red chip background) — warm red, lighter than destructive to distinguish "important" from "dangerous"
- Medium priority: `#FBBF24` (amber-400 text on `#2A2010` dark-amber chip background)
- Low priority: `#9CA3AF` (textSecondary — no chip background, text-only label)
- None: not shown (absence of priority chip = no priority)

**Overdue tasks:** due date text renders in destructive `#EF4444` (not textSecondary). Overdue task rows receive no background change — color on the date chip is the only signal (avoids visual noise).

**Source:** Phase 26 tokens.ts locked values. Priority chip system follows standard task-manager conventions (red/amber/neutral) mapped to the dark palette. Overdue red reuses destructive for semantic consistency.

## Interaction Contracts

### Primary Focal Point

**Phone:** The FAB (56px accent circle, fixed above BottomTabs) is the primary action anchor — it is the first element the eye should land on when the Tasks page loads, and it is the entry point for all task creation.

**Desktop:** The task list column (`TaskListView`, flex-1) is the primary focus area. The list sidebar (200px) is subordinate; the task detail modal overlays the whole page when open.

### Tasks Page

**List navigation (desktop):**
- `TaskListSidebar` is a 200px left panel inside the tasks page content area (not the app-level Sidebar). Inbox is always first. User-created lists follow. Active list has accent `#6366F1` left border (4px) and `#1A1A1A` → slightly lightened `#222222` background.
- Clicking a list loads its tasks in `TaskListView` to the right. No route change per list — state managed via `useState`/`useQueryParam` within `TasksPage`.
- "New list" button at the sidebar bottom opens an inline input (not a modal) for naming the list; pressing Enter or clicking "Create list" saves it.

**List navigation (phone):**
- No sidebar on phone. A list picker lives in a bottom sheet triggered by tapping the current list name header above the task list. Lists Inbox + all user lists; tap selects and dismisses.

**Task row:**
- Tapping the checkbox area (left 44px of row) triggers completion flow.
- Tapping the task title/body opens `TaskDetailSheet`.
- Phone: swipe-left on the row reveals a 72px destructive "Delete" action button (standard iOS swipe pattern). Tapping it triggers the undo toast.
- Desktop: right-click or kebab menu (three-dot icon, shown on row hover, `aria-label="Task options"`) exposes "Edit" and "Delete" actions.
- Minimum row height: 52px (body text + label line + 8px padding top/bottom).

**Quick-add (FAB on phone):**
- Tapping FAB slides up `QuickAddBar` as a bottom sheet, overlaying BottomTabs. Input auto-focuses.
- As the user types, a deterministic client-side parser resolves tokens inline:
  - "tomorrow" / "friday" / "next week" → resolves to a date chip shown below input (e.g. "Fri 19 Jun").
  - `#list-name` → resolved list chip (fuzzy match to existing lists; unmatched → Inbox).
  - `!high` / `!medium` / `!low` / `!1` / `!2` / `!3` → priority chip.
- Pressing "Add task" or Enter on desktop saves the task and clears the input for another entry.
- Escape or tapping outside dismisses without saving.

**Quick-add (keyboard shortcut on desktop):**
- `N` key (when not focused in an input) opens `QuickAddBar` as an inline bar above the task list. Same token parsing as phone.

**Completion micro-animation (TASK-04):**
- When the checkbox is tapped: the circle fills with success green `#22C55E` (150ms ease-out), a checkmark SVG draws in (150ms, stroke-dashoffset animation), then the task row fades and collapses (200ms, max-height 0 with overflow hidden). Total animation: ~500ms.
- Immediately on checkbox tap: start the undo toast countdown (4 seconds).
- During the undo window: task is soft-marked (removed from view but not deleted from Firestore).
- If undo is tapped: reverse the collapse (row re-appears at full opacity, no animation on re-entry), cancel the deletion.
- After 4 seconds without undo: hard-delete the Firestore document.
- For recurring tasks: the next instance is generated server-side during the hard-delete step, then the task list refetches.

**Delete flow:**
- Triggered via swipe action (phone) or kebab "Delete" (desktop).
- No confirmation modal — the undo toast is the only safety net (D-14).
- Same 4-second soft-mark → hard-delete mechanic as completion.
- Undo toast text: "Task deleted." + "Undo" button.

**Undo toast:**
- Positioned: phone = fixed above BottomTabs (bottom: 72px + safe-area); desktop = fixed bottom-center (bottom: 24px).
- Width: fit-content, max-width 320px.
- Background: #1A1A1A, border: 1px solid #2A2A2A, borderRadius: 10px, padding: 12px 16px.
- Text: label size (13px/400) in textPrimary. Action button: accent `#6366F1`, label size, semibold.
- Stacking: if a second completion/delete fires before the first toast expires, extend the countdown for the new item (do not stack multiple toasts — only one active at a time; last action wins).
- Auto-dismiss after 4 seconds with a 200ms fade-out.

**Sort/group control:**
- Rendered as a single row at the top of `TaskListView` (below the list name heading).
- Two segmented button groups: "Sort: Due date / Priority" and "Group: On / Off".
- When "Group" is on: tasks are visually grouped by due date (Today / This week / Later / No date) with a muted section divider label (label size, textSecondary).
- Tapping a sort or group button updates local state only (no URL param needed).

**Task detail sheet:**
- Phone: slides up from bottom as a bottom sheet (same pattern as Phase 26's `DockChat` collapse — CSS transform translateY transition, 250ms ease-out). Drag handle at top. Dismisses by swiping down or tapping outside.
- Desktop: renders as a centered modal (max-width 480px, centered, `#0A0A0A` scrim with `opacity: 0.7`).
- Fields in order: Title (body, single-line), Notes (body, multi-line, 3 rows), Due date picker (native `<input type="date">`), Due time toggle (checkbox to add/remove a time; native `<input type="time">` when enabled), Priority select (None / Low / Medium / High), List selector, Recurrence selector.
- Footer (phone: full-width stacked; desktop: right-aligned row):
  - **New task:** "Add task" button (accent background, 44px height) + "Cancel" link.
  - **Existing task:** "Save changes" button (accent background, 44px height) + "Cancel" link + "Delete task" destructive text button (left-aligned).
- Dirty tracking: changes are not saved until the CTA button is tapped (no auto-save).

**Recurring task edit dialog:**
- When saving an edit to a recurring task, a 2-choice action sheet appears (bottom sheet on phone, inline dropdown on desktop):
  - "This occurrence only"
  - "This and following"
- This choice gates the server action. Tapping outside the sheet cancels the edit.

**Optimistic create/edit:**
- Task creation: new task row appears immediately in the list with a `status: 'saving'` opacity-75 state. On server confirm: full opacity. On server error: row removed + destructive toast "Couldn't save task — try again."
- Task edit: local state updates immediately; API call fires in background. On error: revert to prior values + toast "Couldn't update task — try again."
- Follows the Phase 26 react-query + optimistic update pattern (`useMutation` with `onMutate` / `onError` rollback).

### Today Timeline — Due Tasks Band (D-11)

- Rendered as a new section inside `TimelineDay.tsx`, positioned after all-day calendar events and before timed events.
- Only shown when there are due today or overdue tasks (no empty placeholder for this section — it simply does not render when there are no tasks due).
- Section label: "Due today" in label size (13px), textSecondary, with an accent left-border stripe (4px wide, 32px tall, `#6366F1`).
- Each task in the band renders as a compact row: checkbox (left, 44px tap target), task title (body), and an overdue chip if applicable ("3d overdue" in destructive red).
- Tasks with a due-time (not date-only) that falls within the current hour also appear inline at their time slot in the timed-events section — they appear in both places on that day only.
- Tapping the checkbox triggers the same completion micro-animation as on the Tasks page (undo toast positioned identically).
- Tapping the title row navigates to `/tasks` with that task pre-selected in detail view.

### Glance Rail — Tasks Section (D-12)

- Rendered below the Nutrition section in `GlanceRail.tsx`.
- Shows two lines:
  - "N due today" — textPrimary value, textSecondary label
  - "N overdue" — destructive `#EF4444` value when N > 0, textSecondary label; hidden when N = 0
- Tapping the section navigates to `/tasks` (desktop router navigation, not a link).
- Uses the same `useTaskSummary()` hook that `DueTasksBand` consumes (react-query dedupes the fetch).

**Tone:** Concise, action-first labels. Error messages state the problem and the recovery in one sentence. Overdue formatting is "Nd overdue" (no padding, just the number — "3d overdue", not "3 days overdue") to fit in a compact chip. Plural forms: N is always the raw count, no "1 day" vs "N days" distinction in chips (space-constrained; full form appears in task detail only).

**Task detail CTA states:** The save button is context-aware. When `TaskDetailSheet` is opened for a new task (from FAB or keyboard shortcut), the CTA reads "Add task". When opened by tapping an existing task row (edit mode), the CTA reads "Save changes". Both use the same accent background, 44px height, full-width on phone / right-aligned on desktop.

**Destructive actions:** Delete has no confirmation modal — undo toast is the only recovery (D-14). The "Delete task" button in the detail sheet is styled with destructive `#EF4444` text color (not a filled button) to signal danger without a heavy confirmation step.
