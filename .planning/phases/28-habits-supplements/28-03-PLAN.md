## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HabitStore reads → tick-brain/brain snapshot JSON | habit data enters the autonomous compose path and must json-serialize cleanly |
| habit nudge ↔ 21:30 alert | two crons can address the same supplement; double-nag is the risk |
| HabitStore read → 21:30 prompt render | supplement names flow into a prompt; a serialization or empty-store failure must degrade, not crash |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-doublenag | Denial of Service (nuisance) | autonomous habit nudge + 21:30 alert | mitigate | Per-item-per-day dedup (`habit-nudge:{id}:{date}`, D-17) + distinct topic keys from the fueling-miss alert (D-15); add_topic only after send succeeds (D-10). D-18 keeps Bedtime/pre-bed in the 21:30 lane. |
| T-28-gather-fail | Denial of Service | _gather_habit_adherence / _get_supplement_checkoffs | mitigate | Both return empty sentinels on exception — a HabitStore failure can never break the tick or the 21:30 alert. |
| T-28-arrayunion | Tampering | CoachingTopicStore dedup write | mitigate | Topic keys are plain strings; never a dict-with-SERVER_TIMESTAMP inside ArrayUnion (Pitfall 4). |
| T-28-json | Information Disclosure | habit_pending snapshot in compose | mitigate | get_pending_today returns `_jsonsafe_doc`-coerced dicts (Plan 01); snapshot json.dumps cleanly. |
| T-28-costgate | (cost) | Layer-2 brain | mitigate | habit_pending only flows through Layer 0 (gather $0) → tick-brain (Groq, free) → brain only on affirmative — the existing cost gate is preserved; no new always-on brain call. |
| T-28-SC | Tampering | npm/pip installs | accept | No packages introduced (28-RESEARCH § Package Legitimacy Audit: none). |
