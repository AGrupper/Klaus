<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/habits/* | untrusted client JSON crosses here; validated by Pydantic + auth-gated |
| client `date` field → backfill window | a forged past date would let Amit rewrite locked history |
| client schedule `effective_from` → history | a past effective_from would retroactively rewrite the grid (D-19 break) |
| dose_taken string → rendered UI | unescaped markup could XSS the hub |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-AC | Elevation of Privilege | all /api/habits/* routes | mitigate | `Depends(require_hub_session)` on every route; unauthenticated → 401 (pinned by test_habits_routes_require_session). |
| T-28-backfill | Tampering | POST /api/habits/{id}/checkin | mitigate | Enforce `date ∈ {today, yesterday}` in Asia/Jerusalem; older → 400 (pinned by test_checkin_rejects_day_before_yesterday). |
| T-28-schedule | Tampering | PATCH /api/habits/{id} | mitigate | Reject a schedule revision whose `effective_from < today_iso` → 400 (D-19, pinned by test_patch_schedule_rejects_past_effective_from). |
| T-28-xss | Tampering | dose / dose_taken fields | mitigate | Length-validate on the API (max 200); return as plain strings; frontend renders as React text (never dangerouslySetInnerHTML). |
| T-28-input | Tampering | create/edit bodies | mitigate | Pydantic `Literal`/length/`pattern` constraints on type/slot/days/date (ASVS V5). |
| T-28-hubroutes | Tampering | existing OIDC routes | accept→mitigate | New routes are additive and behind session auth; OIDC `/cron|/internal|/trigger` are not modified (HUB-04). Verified by grep that no OIDC route signature changes. |
| T-28-SC | Tampering | npm/pip installs | accept | No packages introduced (28-RESEARCH § Package Legitimacy Audit: none). |
</threat_model>
