## Component Inventory

### New: HabitRow

Purpose: Single habit/supplement row in the Habits tab list.

Pattern basis: `TaskRow.tsx` (from Phase 27). Simplifications:
- No swipe-to-delete (toggle check-off instead; delete is via kebab only)
- Checkbox is a toggle (checked ↔ unchecked) — filled accent circle when done, open circle when pending
- Supplement type: shows dose label below name in Label (13px) textSecondary
- Slot chip: `#2A2A2A` background, Label (13px) textSecondary, 3px 8px padding, borderRadius 6px
- Streak chip: plain Label (13px) textSecondary inline after slot chip
- Kebab (⋯) menu: "Edit" + "Delete" — same dropdown pattern as TaskRow

Layout per row (minHeight 52px):
```
[CheckButton 44px] [HabitBody flex-1] [KebabButton 32px]
  HabitBody:
    [Name — Body 16px textPrimary]
    [SlotChip] [" · "] [StreakText — Label 13px textSecondary]
    [DoseLabel — Label 13px textSecondary] (supplement only, below slot/streak)
```

### New: HabitsPage

Purpose: Root `/habits` route (currently ComingSoon placeholder in App.tsx).

Phone layout: BottomTabs tabs nav + scrollable list of HabitRows (by slot group: Morning / Noon / Evening / Bedtime) + FAB (phone-only, same 56px accent pattern as TaskFAB).

Desktop layout: matches TasksPage two-column pattern — left sidebar (optional, can start flat) + main content column.

Slot group headers: Label (13px/400) textSecondary, uppercase, 0.04em tracking — same as `FieldLabel` in TaskDetailSheet. No border/divider between groups.

### New: HabitDetailView

Purpose: Per-habit detail showing contribution grid + streak + metadata. Rendered as a bottom sheet (phone) or modal (desktop), same dimensions and slide-in animation as TaskDetailSheet.

Phone: slides up from bottom, 250ms ease-out, drag handle, scrim z:190, sheet z:191.
Desktop: centered modal, max-width 480px, borderRadius 16px.

Contains:
- Habit name (Heading 20px/600)
- Slot chip + streak value (Label 13px)
- Dose field (supplement only, Label 13px)
- Contribution grid (see below)
- History section heading (Label 13px textSecondary uppercase)
- "Edit" and "Delete habit" buttons in the footer

### New: ContributionGrid

Purpose: GitHub-style rolling ~365-day history grid in HabitDetailView.

Grid layout:
- 52 columns (weeks), 7 rows (Mon–Sun from top)
- Each cell: 12px × 12px, borderRadius 2px, gap 2px between cells
- Cell colors: four states (see Color section above)
- Week axis labels: abbreviated month name every ~4 weeks, Label 13px textSecondary
- Day axis: single-letter day initials (M T W T F S S), Label 13px textSecondary

Interaction: cells are read-only in this view (no tap-to-edit past days — backfill is today/yesterday only via check-off toggle). Yesterday's cell shows `pending` fill if not yet checked.

Accessibility: `role="grid"` on the container; each cell `role="gridcell"` with `aria-label="{date}: {state}"`.

### New: HabitCreateEditSheet

Purpose: Create or edit a habit/supplement definition. Same bottom sheet / modal pattern as TaskDetailSheet.

Fields in order:
1. Name (single-line input)
2. Type segmented control: "Habit" | "Supplement" — 2 buttons, full-width, active gets accent background + white text, inactive gets secondary background + textSecondary
3. Dose (text input, only shown when type === "Supplement")
4. Scheduled days (7 day-of-week toggle chips S M T W T F S — pressing All selects all 7, pressing a day toggles it; at least one day required)
5. Time slot segmented control: "Morning" | "Noon" | "Evening" | "Bedtime" — same pattern as Type control

Sheet z-index: scrim z:190, sheet z:191 (beats BottomTabs z:100).
iOS keyboard: useVisualViewport keyboardInset to anchor sheet above soft keyboard.
No autoFocus on phone (iOS layout-pan trap — same rule as TaskDetailSheet).
Scroll-lock: `document.body.style.overflow = 'hidden'` while open.

### New: DoseEditSheet (inline, triggered from timeline or HabitRow check-off)

Purpose: D-09 — when checking off a supplement, show an inline sheet to confirm/adjust the dose before saving.

Triggered only for type === "supplement". For type === "habit", check-off is a single tap with no sheet.

Sheet contains:
- Name + default dose label (Body 16px textPrimary)
- "Adjust dose taken:" label (Label 13px textSecondary uppercase)
- Dose input (prefilled with default dose)
- "Save" button (accent, 44px, full-width on phone)
- "Cancel" text button (textSecondary)

Sheet z-index: z:192 (above HabitDetailView z:191 and HabitsPage sheet).
iOS notes: same keyboard/scroll-lock/no-autoFocus rules as HabitCreateEditSheet.

### New: HabitsBand (Today timeline)

Purpose: TIME-06 — habits and supplements due today appear on the Today timeline.

Pattern basis: `DueTasksBand.tsx` (from Phase 27). The band renders after the DueTasksBand, before timed calendar events, using the same slot-group ordering as HabitsPage (Morning → Noon → Evening → Bedtime → then "any time" items).

Section header: "Habits" label, accent 4px × 32px left-border stripe, 10px 14px padding — identical to DueTasksBand header.

Per-row:
- CheckButton 44px (tap to check off; tap again to uncheck — toggle, D-07)
- Habit name + slot chip (same as HabitRow, condensed single line)
- Dose chip for supplements: Label 13px in border-colored pill ("5g")

On check-off of a supplement: opens DoseEditSheet inline (D-09). On check-off of a habit: immediate toggle.

No navigation on title tap (unlike DueTasksBand which navigates to /tasks) — the habit name is not a link in the band.

### Modified: GlanceRail (desktop)

Add a "Habits" card below the "Tasks" card. Tappable (navigates to /habits).

Structure mirrors "Tasks" card pattern (from Phase 27 GlanceRail):
- Heading "Habits" (20px/600 textPrimary)
- One row per habit with the longest active streak: `[name]` label (13px textSecondary) + `[N]-day streak` value (14px/600 textPrimary)
- Maximum 4 rows to avoid dominating the rail
- Empty: "No habits defined." (13px textSecondary)

### Modified: DueTasksBand → HabitsPage FAB

No conflict. HabitsPage gets its own FAB (phone) matching TaskFAB pattern:
- 56px diameter, accent background, Plus icon, aria-label="Add habit"
- Fixed `bottom: calc(env(safe-area-inset-bottom, 0px) + 76px)`, `right: 16px`
- `zIndex: 40` (same as TaskFAB)
- Wrapper `<div className="md:hidden">` — display driven by class, NOT inline style (Phase 27 UAT lesson)

## Interaction Contracts

### Check-off Toggle (HABIT-02, D-07)

A single tap toggles a habit/supplement between done and pending. No destructive outcome, no undo needed for the check-off itself.

For **habit** type:
1. Tap circle: immediate toggle (API PATCH, optimistic update)
2. On done → unchecked: circle fills accent (#6366F1) for 150ms then reverts to open circle
3. On undone → done: same fill animation in reverse (accent fill fades in, stays)

For **supplement** type:
1. Tap circle: opens DoseEditSheet (not an immediate toggle)
2. DoseEditSheet shows name + prefilled default dose
3. Tap "Save": fires API PATCH with dose-taken value, closes sheet, circle fills accent
4. Tap "Cancel": no state change, sheet closes

### Contribution Grid (HABIT-04, D-13)

Read-only. No interaction on individual cells. The grid scrolls horizontally on phone if the full 52-week range exceeds viewport width (default: overflow-x: auto on the grid container).

Yesterday's cell: if the habit was scheduled yesterday and is still pending (within backfill window), cell renders as `pending` (#2A2A2A) with `aria-label="{date}: pending — can still check off"`). Checking off yesterday's item requires going to the HabitDetailView — not accessible directly from the timeline or HabitRow (timeline/list show only today's items).

Backfill path: HabitDetailView shows yesterday's `pending` cell. Tapping it (if it is yesterday's) opens a DoseEditSheet (supplement) or confirms toggle (habit) with a one-time UI hint: "Marking yesterday — this will repair your streak." This is the only backfill path.

### Delete Habit (D-20)

1. Tap "Delete habit" in sheet footer or kebab menu
2. No confirmation modal — immediate soft-delete
3. Undo toast: "Habit deleted." + "Undo" button (same 4s timer, same UndoToast component, separate undoStore slice for habits vs tasks — or reuse UndoToast with a `type: 'habit'` field)
4. On undo: restore definition AND history (un-delete the Firestore doc)
5. On timer expiry: hard-delete definition + completion log

### Bottom Sheet — iOS Safari Traps (Phase 26/27 lessons, MANDATORY)

All bottom sheets in this phase (HabitCreateEditSheet, HabitDetailView, DoseEditSheet) MUST implement:

1. **z-index beats BottomTabs**: scrim at z:190, sheet at z:191 (BottomTabs is z:100)
2. **No inline `display` on wrappers** — responsive show/hide is driven by Tailwind classes (`md:hidden`, `hidden md:block`), never `style={{ display }}`. This is the root cause of Phase 27 UAT phone/desktop leakage (4 incidents).
3. **Keyboard tracking**: use `useVisualViewport` hook (already in `frontend/src/hooks/useVisualViewport.ts`) to set `bottom: keyboardInset` and `maxHeight: calc(100dvh - {keyboardInset}px - 24px)` — position:fixed does not track the iOS soft keyboard.
4. **Scroll-lock**: `document.body.style.overflow = 'hidden'` while any sheet is open; restore on close.
5. **No phone autoFocus**: do NOT set `autoFocus` on any input inside a phone bottom sheet — iOS pans the layout viewport mid-slide animation (produces the left-shift bug). Desktop can use `autoFocus`.
6. **Blur-before-click trap**: for any button that dismisses the sheet (Cancel, scrim tap), use `onMouseDown={(e) => e.preventDefault()}` to prevent iOS from firing blur on the active input before the click fires — otherwise blur eats the submit. See Phase 26 DockChat pattern for reference.

### Completion Animation (on check-off)

Mirrors task completion animation from Phase 27 (TaskRow.tsx), adapted for toggle behavior:

- **Done state**: circle fills solid accent `#6366F1`, duration 150ms ease-out, shows a checkmark SVG overlay (opacity 0→1, 150ms)
- **Undone state**: accent fill fades out to empty circle, checkmark fades out, duration 150ms
- **No row collapse** (unlike tasks) — the habit row stays visible after check-off (it remains on the list)
- Touch target for the check circle: 44px × 44px centered hit area

## Responsive Layout

### Phone (< 768px — md breakpoint)

- HabitsPage: full-screen scrollable list, BottomTabs at bottom, FAB fixed above tabs
- Slot group headers: sticky at top of each group as user scrolls (position: sticky, same `#0A0A0A` background as dominant to avoid bleed)
- HabitDetailView: bottom sheet (same as TaskDetailSheet phone mode)
- HabitCreateEditSheet: bottom sheet
- ContributionGrid: horizontally scrollable (overflow-x: auto), shows ~12 weeks visible by default
- Habits band in Timeline: renders inline, same padding as DueTasksBand (`14px` sides)
- DoseEditSheet: bottom sheet, z:192

### Desktop (≥ 768px — md breakpoint)

- HabitsPage: two-column layout (matches TasksPage): optional left sidebar (can be added later, start flat with single column), main habits list column
- GlanceRail: add "Habits" card below "Tasks" card, navigate to /habits on click
- HabitDetailView: centered modal, max-width 480px, `border-radius 16px`, `#0A0A0A` scrim 70% opacity
- HabitCreateEditSheet: centered modal, same dimensions
- ContributionGrid: renders full 52-week width (fits ≥ 480px modal)
- DoseEditSheet: centered modal, z:202 (above HabitDetailView z:191 + RecurringScopeSheet-equivalent)

**Invariant**: Every wrapper whose visibility differs between phone and desktop MUST use a Tailwind class for the `display` property, never `style={{ display }}`.
