<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API JSON → React render | dose/dose_taken and habit name strings are rendered into the DOM |
| phone vs desktop wrappers | inline display leaks phone-only UI to desktop (Phase 27 UAT, 4 incidents) |
| iOS Safari bottom sheets | keyboard/z-index/blur traps eat submits or hide the sheet |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-xss | Tampering | dose / dose_taken / name render | mitigate | Render as plain React text only; no `dangerouslySetInnerHTML` (grep gate in Task 2/3). API also length-validates (Plan 02). |
| T-28-display | (UX correctness) | responsive wrappers | mitigate | All show/hide via Tailwind classes (`md:hidden`); no inline `style={{display}}` (grep gate, Pitfall 2). |
| T-28-iossheet | (UX correctness) | create/edit + dose sheets | mitigate | z-index 190/191/192, useVisualViewport keyboardInset, scroll-lock, no phone autoFocus, onMouseDown preventDefault on dismiss (Pitfall 3). |
| T-28-auth | Elevation of Privilege | all /api/habits/* calls | mitigate | apiFetch sends the session cookie; routes gated server-side (Plan 02). No token handling in the client. |
| T-28-SC | Tampering | npm installs | accept | No new npm packages (28-RESEARCH § Package Legitimacy Audit: none; package.json already has react-query/zustand/lucide). |
</threat_model>
