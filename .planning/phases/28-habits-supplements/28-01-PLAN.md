<objective>

(two collections — `habits/{id}` definitions with append-only `schedule_history`, and
`habit_completions/{YYYY-MM-DD}/records/{habit_id}` daily log recording dose-taken) plus

(D-19), and the DST-safe date handling correct here is the whole technical risk of the phase.

HabitStore (memory/firestore_db.py):
  _COLLECTION = "habits"; _COMPLETIONS = "habit_completions"
  create(habit: dict) -> dict                          # returns doc without updated_at
  get(habit_id: str) -> dict | None
  list_active() -> list[dict]                           # status == "active"
  update(habit_id: str, fields: dict) -> dict | None    # PATCH; schedule edit appends to schedule_history
  soft_delete(habit_id: str) -> None                    # status -> "completing"
  restore(habit_id: str) -> None                        # status -> "active" (undo)
  delete(habit_id: str) -> None                         # hard-delete def + all completion records for this habit
  log_completion(date_str, habit_id, done, dose_taken=None) -> None   # idempotent toggle; done=False deletes record
  get_completions_for_date(date_str: str) -> dict[str, dict]          # {habit_id: completion_doc}
  get_pending_today(today_iso: str) -> list[dict]       # scheduled-today, not-done items + current streak
  get_history(habit_id: str, today_iso: str, window_days=365) -> dict # {"streak": int, "grid": [{date,state}]}
  get_summary(today_iso: str) -> dict                   # {"pending_today": int, "streak_leaders": [{id,name,streak}]}

compute_streak_and_grid(habit_id, schedule_history, completions, today, window_days=365) -> dict
  # pure function; today is a datetime.date; completions keyed by "YYYY-MM-DD"
  # returns {"streak": int, "grid": [{"date": "YYYY-MM-DD", "state": "done"|"missed"|"not-scheduled"|"pending"}]}

Definition doc shape (habits/{id}):
  id, name, type ("habit"|"supplement"), dose (str|None), slot ("Morning"|"Noon"|"Evening"|"Bedtime"),
  schedule_history: [{"effective_from": "YYYY-MM-DD", "days": "daily" | [int Mon=0..Sun=6]}],
  status ("active"|"completing"), created_at (ISO str), updated_at (SERVER_TIMESTAMP)

Completion doc shape (habit_completions/{date}/records/{habit_id}):
  habit_id, date ("YYYY-MM-DD"), done (True), dose_taken (str|None), logged_at (ISO str), updated_at (SERVER_TIMESTAMP)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → HabitStore writes | habit definitions + completion records originate from `/api/habits/*` (Plan 02), behind `require_hub_session` — but the store itself must not corrupt history on schedule edits |
| Firestore read → json.dumps | `DatetimeWithNanoseconds` from `SERVER_TIMESTAMP` reads must not leak into any JSON response |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-schedule | Tampering | `HabitStore.update` schedule edit | mitigate | Forward-only: append a new `schedule_history` revision with `effective_from = today_iso`; never mutate prior revisions (D-19 / Pitfall 7). The PATCH-time date floor is additionally enforced in Plan 02. |
| T-28-streak | Tampering | `compute_streak_and_grid` | mitigate | Pure reset on confirmed miss; pending never breaks; non-scheduled neutral — pinned by `TestStreakComputation` + `TestGridDerivation` so the algorithm cannot silently regress. |
| T-28-dst | Tampering | `compute_streak_and_grid` date math | mitigate | Date-only (`datetime.date`) arithmetic — DST transitions invisible; pinned by spring-forward/fall-back fixtures. |
| T-28-json | Information Disclosure | HabitStore reads | mitigate | `_jsonsafe_doc` on every `snap.to_dict()` (Pitfall 1); strips `updated_at` before serialization. |
| T-28-SC | Tampering | npm/pip installs | accept | No packages introduced this plan (28-RESEARCH § Package Legitimacy Audit: none). |
</threat_model>
