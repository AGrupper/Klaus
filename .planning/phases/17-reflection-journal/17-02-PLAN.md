## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gathered day-data → brain LLM | best-effort metrics + conversation text crosses into an LLM prompt |
| brain LLM output → Firestore/Pinecone | LLM-generated JSON is parsed then stored verbatim |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-04 | Tampering | `_parse_reflection_json` of brain output | mitigate | Hardened parse (Task 2): strip fences, slice `{`..`}`, validate all 5 keys/types, default missing. LLM output is never `eval`'d/executed — only stored. Malformed output falls through to the D-13 minimal fallback rather than corrupting `self_state` |
| T-17-05 | Denial of Service | per-source gather + LLM calls | mitigate | D-01 per-source try/except (Task 2: ≥5 `except Exception` blocks) + D-13 brain/fallback chain → a failing source or a total LLM outage cannot abort the run or leave a gap in the journal |
| T-17-06 | Information Disclosure | conversation read in cron context | accept | The 6h `SESSION_TIMEOUT_HOURS` window naturally limits what a 22:00 cron reads; we do NOT bypass the gate to read raw `conversations/{user_id}`. Single-user trusted system — LOW severity |
