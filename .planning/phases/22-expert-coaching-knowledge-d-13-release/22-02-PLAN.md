## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| brain tool args → `read_coaching_guide(topic)` | `topic` is an LLM-supplied string. It must resolve to an authored section anchor, never to a filesystem path. |
| coaching-guide file → system prefix | Slim core file content is injected verbatim into every brain system prompt. |
| brain ↔ worker tool partition | `read_coaching_guide` must remain brain-direct; the worker must never gain it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-04 | Information Disclosure / Path Traversal | `_handle_read_coaching_guide(topic)` | mitigate | `topic` is normalized to a slug and used ONLY inside a regex against authored `<!-- SECTION: slug -->` anchors in a hardcoded file path. It is NEVER concatenated into a filesystem path. `..`, `/`, absolute paths cannot escape — they simply fail to match a slug and return error JSON. Test `coaching_guide_unknown_topic` exercises the no-match path. |
| T-22-05 | Elevation of Privilege | worker gaining `read_coaching_guide` | mitigate | Tool added to `SMART_AGENT_DIRECT_TOOLS` and to the `WORKER_TOOL_SCHEMAS` exclusion set; `test_read_coaching_guide_not_in_worker_schemas` asserts the worker cannot see it. Honors CLAUDE.md "brain never routes through worker first". |
| T-22-06 | Denial of Service | oversized slim-core injection on every call | mitigate | `_load_coaching_guide_slim` extracts only the marker block; >10000 chars logs a warning; the < 15000-char unit test gates it. Prevents the whole 1000+ line guide leaking into every prompt (Pitfall 2). |
| T-22-07 | Tampering | missing/malformed guide file at runtime | accept | Loader returns "" with a warning on OSError/missing markers — coaching injection degrades gracefully rather than crashing the orchestrator. Low risk; documented. |
| T-22-SC | Tampering | npm/pip/cargo installs | n/a | No package installs in this plan (uses stdlib `re`/`json`/`pathlib` already imported). |
</threat_model>
