## Transport mechanism

| Option | Description | Selected |
|--------|-------------|----------|
| iOS Shortcuts (free) | Native, zero cost, scheduled or app-close triggers. Aligns with autonomous-tick cadence. | ✓ |
| Health Auto Export | Paid (~$5/mo). True on-write trigger, mature webhook bridge. | |
| Both (Shortcuts first, HAE later) | Ship Shortcuts now, leave Klaus side source-agnostic so HAE can be swapped in. | |

**User's choice:** iOS Shortcuts.
**Notes:** User also asked "is there a way for it to be every time I close the Lifesum app the webhook runs?" — confirmed yes, iOS Personal Automation "When App is Closed" trigger (iOS 15+) with "Run Without Asking" toggle (iOS 15.4+ default).

## Sample window per fire

| Option | Description | Selected |
|--------|-------------|----------|
| Last 24 hours | Simple, robust, idempotent. If close-trigger misses, next fire still catches it. ~10–20 samples. | |
| Last 2 hours | Smaller payload, but a missed trigger means a missed meal until next fire. | ✓ |
| Since last successful sync | Most efficient, but requires watermark persistence on the iPhone. | |

**User's choice:** Last 2 hours.
**Notes:** Trade-off acknowledged — missed close-trigger + locked-phone moment could drop a meal until the 23:55 daily catch-up rescues it. MealStore upsert is idempotent so re-pushes are free; user accepts the trade-off in exchange for a smaller per-fire payload.

## Safety net for missed close-trigger fires

| Option | Description | Selected |
|--------|-------------|----------|
| No safety net | Trust the close-trigger. Missed fires drop silently. | |
| Daily 23:55 catch-up | Second Personal Automation runs at 23:55 with 24h lookback. Idempotent. End-of-day worst case. | ✓ |
| Hourly catch-up | Every hour with 2h window. Maximum reliability, 24 fires/day. | |

**User's choice:** Daily 23:55 catch-up.

## Webhook auth model

| Option | Description | Selected |
|--------|-------------|----------|
| Shared-secret bearer | Long random token in Secret Manager, pasted into Shortcut Authorization header. | ✓ |
| HMAC-signed payload | Shortcut computes HMAC-SHA256 over body. Tamper-evident; more work to build in Shortcuts. | |
| Bearer + IP allowlist | Defense in depth; IPs change with travel/cellular. | |

**User's choice:** Shared-secret bearer.
**Notes:** New helper `_verify_healthkit_request` mirrors `_verify_cron_request` shape. Secret name: `klaus-healthkit-webhook-token`. Rotation via standard `gcloud secrets versions add` flow.

## Endpoint behaviour on receipt

| Option | Description | Selected |
|--------|-------------|----------|
| Upsert only, next tick decides | Webhook normalizes → MealStore.upsert → 200. Next */20 tick handles judgment via existing wiring. Preserves eval coverage. | ✓ |
| Upsert + trigger inline tick | Webhook upserts and synchronously calls run_autonomous_tick. Near-real-time nudge. Second tick entry point to manage. | |
| Upsert + background-task trigger | Webhook upserts, FastAPI BackgroundTask fires tick ~1s later. Concurrency complexity. | |

**User's choice:** Upsert only — next tick decides.
**Notes:** Preserves all 5 eval fixtures' coverage. Worst-case nudge latency ≈20 min. No new judgment paths.
