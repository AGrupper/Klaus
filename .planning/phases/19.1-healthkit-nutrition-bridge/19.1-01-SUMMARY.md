## Key fixture findings (informs Plan 02)

The fixture went through two capture iterations on the same operator session — the first run (`Find Health Samples` with `Limit: 1`) hit a phantom 0-quantity sample at the Shortcut's run time, so all macros came back zero. After turning `Limit` off, the Shortcut returned real Lifesum samples but mashed them as newline-concatenated text (the production Shortcut needs a `Repeat With Each` loop to emit per-sample dicts — deferred to Plan 05's Shortcut runbook). The operator transcribed the real daily totals from the Apple Health app (1,038 kcal / 109g carbs / 70g protein / 38g fat) — those are the values now locked in the fixture. Fiber 5.6g came from the second capture.

1. **`uuid: "Lifesum"`** — `Get Details of Health Samples → Source` returns the literal source-app name string. This is *not* the HKObject UUID. **Implication for Plan 02:** every empty/identical-source sample would collapse to `source_id = "healthkit:Lifesum"` in production, breaking dedup. **Decision locked (operator-confirmed):** normalizer generates deterministic fallback `source_id = f"healthkit:{start_date_iso}:{calories_int}"` when `uuid` is empty OR matches a known source-name (e.g. "Lifesum"). Plan 02 must implement this fallback path and test for it.
2. **`start_date: "2026-05-29T07:30:00GMT+3"`** — non-ISO-8601 timezone suffix. **Implication for Plan 02:** Pydantic V2's default datetime parser may reject `GMT+N`. **Action:** Plan 02 must add a `@field_validator("start_date", mode="before")` that normalizes `GMT+N` → `+0N:00` (or hands off to `dateutil`). Test with both `+03:00` and `GMT+3` literal inputs.
3. **`samples_by_type` numerics arrived as `int`/`float` here, not strings** — but the broader RESEARCH.md Q1 finding (Shortcuts often emits stringy numerics) still applies. **Action:** Plan 02's `test_string_numerics_coerce` should still exist; build a mock dict to exercise the coercion path since the fixture alone won't.
4. **`metadata` not present.** Lifesum did not write `HKMetadataKeyMealTime`. **Implication for Plan 02:** hour-bucket fallback (D-13) is the primary code path; metadata-override is best-effort. Matches RESEARCH.md Q2.
5. **`DietaryFiber_g` is present** alongside the 4 required macros. The normalizer must silently ignore unknown `samples_by_type` keys (via `.get(..., 0)` per CONTEXT.md `<specifics>`).
6. **Shortcut design needs `Repeat With Each` loop** to emit proper multi-sample JSON. The Wave 0 capture used Set-Variable + Text-template which only works for single-sample output. **Action for Plan 05:** the operator runbook in `docs/healthkit_shortcut.md` must include the Repeat-With-Each loop step-by-step so the production Shortcut emits a true `[{...}, {...}]` array.
