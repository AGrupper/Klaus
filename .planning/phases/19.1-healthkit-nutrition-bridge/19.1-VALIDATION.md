## Security Domain (ASVS)

Aligns with `security_enforcement` default-on. Threat model lives in PLAN.md `<threat_model>` block (planner injects).

| ASVS Category | Applies | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | yes | Shared-secret bearer (D-06); `hmac.compare_digest` constant-time; ≥32-byte token entropy (`secrets.token_urlsafe(32)`) |
| V3 Session Management | no | Stateless webhook |
| V4 Access Control | yes | Single token = single principal (Amit's iPhone); no role hierarchy |
| V5 Input Validation | yes | Pydantic model (D-15); FastAPI auto-422 |
| V6 Cryptography | yes | TLS via Cloud Run; no app-layer crypto (HMAC declined per D-06) |
| V7 Error Handling | yes | Log auth failures with redacted token prefix; never return token in error body |
| V8 Data Protection | yes | Token in Secret Manager (`klaus-healthkit-webhook-token`); never logged plaintext |
