<objective>
Add three read-only aggregator routes — `GET /api/health/training`,
`GET /api/health/nutrition`, `GET /api/health/sleep` — under `/api/*`, each gated by
`require_hub_session`, each mirroring the `api_today()` composition pattern
(`run_in_executor` + `asyncio.gather` + `_jsonsafe_doc`). Every number a chart will
draw (weekly buckets, averages, gap markers, targets) is computed server-side so the
frontend renders pre-aggregated `{x, y}` points with zero client-side math.

Purpose: These are the data contracts the three health pages consume. They must uphold
the project's server-computes/client-renders invariant (the 2026-06-09 drifting-numbers
lesson), the slot-label-never-eating-time invariant, and event-loop safety (the
2026-06-24 weekly-review-500 incident class).
Output: 3 routes + private `_health_*` helpers + a `_resolve_range` allowlist +
3 Wave-0 integration test files.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/health/* | Untrusted `range` query param crosses into date arithmetic |
| unauthenticated → /api/* | Session cookie required; must reject on absence |
| route → Postgres/Firestore | Aggregation of personal health data; must not leak stack traces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-02-01 | Tampering | `range` query param | mitigate | `_resolve_range` allowlist `{7d,30d,90d,1y}`→fixed int; never `int()`-parse arbitrary client input into `timedelta` (RESEARCH.md Security V5) |
| T-30-02-02 | Spoofing | all 3 routes | mitigate | `Depends(require_hub_session)` on every route; 401 test per route (HUB-01, HUB-04 — OIDC routes untouched) |
| T-30-02-03 | Denial of Service | nutrition 1y read | mitigate | Bounded by allowlist (max 365); reuse `_routes_cache` TTL shape if needed; executor-wrapped so event loop is not starved (Pitfall 1/3) |
| T-30-02-04 | Information Disclosure | slot timestamps | mitigate | No slot_time/HH:MM field emitted; slot LABELS only (CLAUDE.md §6 invariant, D-13/D-16) — asserted by test |
| T-30-02-05 | Information Disclosure | error paths | mitigate | Per-source helpers try/except → safe default + WARNING log; no raw traceback in JSON response |
| T-30-02-SC | Tampering | npm/pip installs | accept | No new packages this phase (RESEARCH.md audit: not applicable) |
</threat_model>
